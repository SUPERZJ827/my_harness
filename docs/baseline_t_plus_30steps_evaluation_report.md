## 分析

### 1. 提高到 30 steps 后，`baseline_t_plus` 整体是有增益的

最直接的变化是：

- `Avg CR` 提升 `+4.10` 个百分点
- `Pass@1` 提升 `+10` 个百分点
- 跑通题数从 `10/20` 增加到 `11/20`

这说明 `T_plus` 在原本 20 steps 的设置下，至少对一部分任务存在预算偏紧的问题。把步数放宽以后，它不是“完全脱胎换骨”，但确实获得了稳定的整体提升。

### 2. 最大受益来自 DL 与部分 CSV 任务

本次最明显的改善来自：

- `dl_16`：`0.000 -> 1.000`
- `csv_excel_48`：`0.000 -> 0.333`
- `csv_excel_39`：`0.500 -> 0.636`
- `csv_excel_32`：`0.750 -> 0.875`

同时：

- `DL Avg CR` 从 `15.83%` 提升到 `35.83%`
- `DL Pass@1` 从 `0.00%` 提升到 `20.00%`

这说明把 `T_plus` 的步数预算从 20 提高到 30，对原本偏弱的 DL 子集是有帮助的。

### 3. 但提升并不是单调正向，Human 子集有回落

虽然总体更好了，但 `human` 子集并没有同步提升：

- `Human Avg CR` 从 `59.44%` 下降到 `52.22%`

其中下降最明显的是：

- `human_12`：`0.750 -> 0.125`
- `human_11`：`0.667 -> 0.167`
- `human_7`：`0.667 -> 0.333`

这说明“放宽 steps”虽然能帮到一部分长链任务，但也可能让某些原本较快收敛的人类分析类任务出现更多偏移或额外错误。

### 4. `T_plus` 的核心收益仍然更像治理与对齐，而不是纯推理增强

本次 30 steps 版本依然保留了 `T_plus` 的原有特征：

- `CSV Avg CR` 很强，达到 `66.86%`
- `F5` 没变，仍为 `35.00%`
- `F2` 仍是 `0.00%`

这说明步数放宽主要改变了“完成更多链路”的机会，但没有改变 `T_plus` 的本质角色。它依旧更像是：

- 对结构化输出和 artifact 契约有帮助
- 对部分建模结果有促进
- 但不是一个万能的 reasoning 增强层

## 结论

对 `baseline_t_plus` 来说，把 `max_steps` 从 `20` 提高到 `30` 是值得的，因为它带来了明确的总体收益：

- 更高的 `Avg CR`
- 更高的 `Pass@1`
- 更好的 `DL` 与 `CSV` 表现

但这个改动不是无代价的。它在带来总体提升的同时，也让部分 `human` 任务退化。因此当前更准确的结论是：

- `30 steps` 比 `20 steps` 更适合作为 `baseline_t_plus` 的候选设置
- 但它还不能说明“只要继续增加步数就一定全面变好”
- `T_plus` 的收益依然是结构化、任务相关的，而不是所有子集上均匀增长
